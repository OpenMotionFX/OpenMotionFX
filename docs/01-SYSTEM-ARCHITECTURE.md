# 01 — System Architecture

## 1. Topographical Split (Brain vs. Brawn)
OpenMotionFX separates processing complexity from power delivery via a System-on-Module pattern[cite: 1].
```
┌────────────────────────────────────────────────────────────┐
│             CORE TRACKING MODULE (The Brain)               │
│  ESP32-S3  │  DWM3000 UWB  │  BNO085 IMU  │  TP4056 Charge │
└──────────────────────────┬─────────────────────────────────┘
                           │
                           │- High-Density Mezzanine Interconnect
                           │
                           ▼
┌────────────────────────────────────────────────────────────┐
│             CARRIER HATS (The Brawn)                       │
│   [Lighting Hat Profile]      OR     [Motion Hat Profile]  │
│   Constant-Current Drivers           Trinamic TMC Drivers  │
│   MOSFET Arrays                      AS5600 Encoders       │
└────────────────────────────────────────────────────────────┘
```

## 2. Hardware Auto-Detection Architecture
The application profiles are completely plug-and-play[cite: 1]. One universal firmware binary runs on the Core Module[cite: 1]. The destination profile is parsed during the boot sequence by evaluating an Analog-to-Digital Converter (ADC) pin connected to a voltage divider on the target Carrier Hat[cite: 1].

* **No Carrier Hat (Standalone Anchor/Subject Tag):** ADC pin reads 0V. Firmware launches standard tracking beacon profiles[cite: 1].
* **Lighting Carrier Hat:** Resistor array forces a drop to 1.0V on the target pin. Firmware spins up sACN/Art-Net network tasks and maps output to lighting control registers[cite: 1].
* **Motion Carrier Hat:** Resistor array forces a drop to 2.0V on the target pin. Firmware initializes SPI/UART communication with Trinamic drivers and monitors endstop interrupts[cite: 1].

## 3. Spatial Resolution Tracking Engine
Sub-centimeter physical accuracy is achieved by combining four distinct radio and sensor processing techniques:

### A. Phase Difference of Arrival (PDoA)
The Core Module features two UWB antennas arrayed exactly half a wavelength apart[cite: 1]. By measuring the fractional picosecond arrival variations between the antennas, the system evaluates the absolute Angle of Arrival (AoA) in addition to Time of Flight distance spheres[cite: 1].

### B. Channel Impulse Response (CIR) First-Path Detection
To eliminate RF multipath reflections generated by metal C-stands, tracking rails, and set structures, the firmware bypasses the highest amplitude signal peak[cite: 1]. It directly monitors the CIR time-domain log to catch the earliest, direct line-of-sight signal path[cite: 1].

### C. Double-Sided Two-Way Ranging (DS-TWR)
To eliminate clock drift caused by differences in cheap crystal oscillators across microcontrollers, ranging checks employ a multi-step handshake[cite: 1]. Node A pings Node B, Node B replies, Node B pings Node A, and Node A provides the final response[cite: 1]. The mathematical combination cancels out transceiver clock errors[cite: 1].

### D. Tightly Coupled Error-State Kalman Filter (ESKF)
UWB spatial coordinates sample at roughly 20-50Hz[cite: 1]. The BNO085 IMU outputs internal sensor-fused orientation data at 400Hz+[cite: 1]. The master application executes an ESKF: the high-rate accelerometer data propagates the spatial state between radio cycles, while the UWB mesh readings bound and correct the IMU drift[cite: 1].
