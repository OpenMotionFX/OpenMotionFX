# 01 — System Architecture & Physical RF Mechanics

## 1. Topographical Split (Brain vs. Brawn)
OpenMotionFX isolates compute and positional awareness away from high-power line-switching components through a modular System-on-Module (SoM) architecture.

```
┌────────────────────────────────────────────────────────────┐
│             CORE TRACKING MODULE (The Brain)               │
│  ESP32-S3  │  DWM3000 UWB  │  BNO085 IMU  │  TP4056 Charge │
└──────────────────────────┬─────────────────────────────────┘
                           │
                           │ High-Density Mezzanine Interconnect
                           │
                           ▼
┌────────────────────────────────────────────────────────────┐
│             CARRIER HATS (The Brawn)                       │
│   [Lighting Hat Profile]      OR     [Motion Hat Profile]  │
│   Constant-Current Drivers           Trinamic TMC Drivers  │
│   MOSFET Arrays                      AS5600 Encoders       │
└────────────────────────────────────────────────────────────┘
```

## 2. Hardware Auto-Detection Architecture
A single firmware binary runs universally across all deployed Core Modules. The operational profile is dynamically mapped at boot time by sampling the voltage drop generated by a precision resistor network built into the host Carrier Hat.

* **Standalone (No Hat Connected):** ADC reads $0\text{ V}$. Node runs as a standard tracking Anchor or mobile Subject Tag.
* **Lighting Hat Profile:** Resistors output $1.0\text{ V}$ to the ADC pin. Node spins up sACN/Art-Net background listeners and enables constant-current output arrays.
* **Motion Hat Profile:** Resistors output $2.0\text{ V}$ to the ADC pin. Node mounts Trinamic communication loops, maps endstop interrupts, and watches AS5600 encoder streams.

## 3. Ad-Hoc Mesh & Anchor Self-Calibration
The system eliminates manual room measurement through automated geometric reconstruction.

### A. The Discovery & Inter-Ranging Phase
When the system transitions to calibration mode, the 4+ randomly placed hardware Anchors execute an isolated round-robin ranging routine. Every anchor pings every other anchor thousands of times per second using UWB pulses to calculate a matrix of absolute scalar distances.

### B. Multidimensional Scaling (MDS) Engine
The calculated distance matrix passes to the Orchestrator, which executes a Multidimensional Scaling (MDS) algorithm (or Distributed Gradient Descent). By minimizing stress metrics across the relative coordinate pairs, the engine mathematically reconstructs the absolute 3D position of every tracking anchor within ~3 seconds.

### C. Relative Global Origin Snapping
The reconstructed 3D grid is initially arbitrary. To anchor the coordinate system to the practical film set, the software benchmarks the initial coordinates of the **Subject Node** (attached to the actor or primary focus element) at execution startup. The entire virtual canvas shifts via translation vectors so that:

$$
\begin{bmatrix} X_{\text{render}} \\ 
                Y_{\text{render}} \\ 
                Z_{\text{render}}
\end{bmatrix} = 
\begin{bmatrix} X_{\text{mesh}} \\
                Y_{\text{mesh}} \\ 
                Z_{\text{mesh}} 
\end{bmatrix} - 
\begin{bmatrix} X_{\text{origin}} \\
                Y_{\text{origin}} \\
                Z_{\text{origin}} 
\end{bmatrix}
$$

This establishes a relative origin of $(0,0,0)$ at the subject's exact baseline location, making floor irregularities or room symmetry irrelevant to the virtual environment.

## 4. Advanced RF Physics & Sub-Centimeter Resolution Control
Standard commercial UWB tracking platforms are bounded to a $\pm 10\text{ cm}$ resolution limit. OpenMotionFX passes this barrier to hit sub-centimeter physical accuracy by implementing four advanced radio-frequency and sensor-fusion mechanisms:

### Trick A: Phase Difference of Arrival (PDoA) Angle Extraction
Standard systems evaluate distance as a sphere using purely time-of-flight variables. OpenMotionFX utilizes two distinct receiver antennas on the Core Module PCB spaced exactly at:
$$d = \frac{\lambda}{2}$$
At a carrier frequency of $6.5\text{ GHz}$, this spatial separation is approximately $2.3\text{ cm}$. When the incoming radio wave front strikes the node, it hits one antenna a fraction of a picosecond before the second. By monitoring the phase difference of the arriving wave, the DWM3000 chip computes the exact **Angle of Arrival (AoA)**. Every single ping yields both a distance vector and an angular vector, locking down high-resolution 3D positioning even in non-linear rooms.

### Trick B: Channel Impulse Response (CIR) First-Path Separation
Physical film environments contain extensive metallic structures (aluminum sliders, steel C-stands, lighting panels) that generate severe RF multipath reflections. The strongest signal received by a tracking node is frequently an indirect reflection bouncing off a wall or floor.

```
                   Reflected Path (High Amplitude)
              ┌─────────────────────────────────────────┐
              ▼                                         │
        [Anchor Node] ─────────────────────► [Subject Module Antennas]
                           Direct Path

```
The firmware taps directly into the DWM3000's internal Channel Impulse Response (CIR) register arrays. Instead of monitoring peak signal amplitude, the ESP32-S3 parses the raw time-domain accumulation curve, isolates the **earliest rising edge** (the true direct line-of-sight signal), and discards all subsequent multi-path energy peaks.

### Trick C: Clock-Drift Cancellation via DS-TWR
Microcontrollers depend on low-cost internal crystal oscillators that display independent micro-variations. If a tracking node's hardware clock runs even $0.0001\%$ faster than an anchor's clock, time-of-flight math drifts by feet. The firmware enforces **Double-Sided Two-Way Ranging (DS-TWR)**:

1. Node A transmits a Poll message to Node B.
2. Node B records the arrival time and returns a Response packet.
3. Node A records the response arrival time and transmits a final Final message.
4. Node B tracks the execution window to construct the complete propagation algorithm.

The absolute time-of-flight distance calculation uses the following formulation:

$$T_{\text{prop}} = \frac{(T_{\text{round1}} \times T_{\text{round2}}) - (T_{\text{reply1}} \times T_{\text{reply2}})} {T_{\text{round1}} + T_{\text{round2}} + T_{\text{reply1}} + T_{\text{reply2}}}$$

This matrix distribution completely cancels out the linear clock errors of both independent microcontrollers, stabilizing tracking precision down to the millimeter tier.

### Trick D: Tightly Coupled Error-State Kalman Filter (ESKF)
UWB positional telemetry runs at a fixed, radio-limited update rate of $20\text{ Hz}$ to $50\text{ Hz}$. This update frequency is too slow and stepped for direct cinema cameras or fluid virtual production tracking. The Core Module addresses this by pulling tracking data from the BNO085 IMU's onboard sensor fusion array at $400\text{ Hz}$.
